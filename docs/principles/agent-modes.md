# Agent Modes

Founderos operates in two public modes: a conversational clarification mode and an execution mode.

These modes exist to separate discussion from implementation and to keep execution bounded.

## Conversational clarification mode

This is the default mode.

In this mode, Founderos helps clarify:

- goals
- constraints
- tradeoffs
- priorities
- next steps

The purpose of this mode is understanding. The system helps the founder think, compare options, and make the work legible before anything moves toward execution.

## Execution mode

Execution mode begins when the founder clearly asks for implementation.

At that point, Founderos shifts from open-ended discussion to concrete handoff. The objective is not expansive reasoning for its own sake. The objective is to express the next work in a bounded, truthful, implementable form.

## Why the distinction matters

Without a stable distinction between clarification and execution, systems tend to drift into ambiguity:

- discussion starts to masquerade as commitment
- rough ideas start to look like approved actions
- implementation intent becomes harder to inspect

Founderos keeps the boundary explicit so execution can remain deliberate and reviewable.

## Stable public rule

The practical rule is simple:

- conversation is for clarifying what should happen
- execution is for specifying what will be implemented next

That distinction helps preserve human authority while still making the system useful as an execution partner.
