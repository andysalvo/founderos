## Summary

Introduce a formal governance layer before admitting the first OpenClaw external tool.

## Context

Founderos has reached the point where adding OpenClaw tooling is operationally attractive and strategically important. However, doing so without a formal governance layer risks widening practical agency without a durable statement of authority, boundaries, amendment process, widening standards, and retrenchment discipline.

The repo already contains boundaries, principles, and decision structures, but they do not yet exist as a single constitutional layer that can support future agentic workflow growth.

## Options considered

1. add tools first and document later
2. rely on existing boundaries docs without a formal constitution
3. write a constitution and initial amendments before tool admission

## Decision

Write and adopt a governance layer before admitting the first external OpenClaw tool.

## Rationale

This preserves the original feel of Founderos as a bounded, founder-centered, governance-aware system while making future capability growth explicit rather than implicit. It also allows the system to work toward broader agentic workflow quality without requiring each new capability to reinvent first principles.

## Consequences

- governance becomes easier to interpret
- future tool additions gain a standard admission path
- repo readers are less likely to misunderstand Founderos as unconstrained agency
- orchestration growth can occur without dissolving APS authority
- constitutional drafting becomes part of the path to capability growth

## Follow-up

- finalize research corpus
- finalize constitution
- finalize initial amendments
- update README to make governance required reading
- use the amendment path for first external tool admission
